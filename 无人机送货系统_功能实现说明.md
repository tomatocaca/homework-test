## 1. 系统概述

无人机送货系统是一个 C++ 控制台程序，模拟校园内的无人机快递配送。系统包含**三种角色**：管理员、配送员、普通用户，每种角色通过账号密码登录后进入各自的操作界面。

核心功能：

- 用户在四个站点间下单
- 配送员分配无人机执行配送
- 管理员管理无人机和查看统计报告
- 每架无人机每日最多配送 **5次**
- 用户可以提前 **3天** 预约并选择时段
- 一周取消 **3次** 订单将被封号 **2周**

### 2.3 登录流程

通用登录函数 `login(role)` 接收角色类型参数，提示用户输入账号密码，然后遍历全部账户列表进行**三重比对**（账号 + 密码 + 角色），匹配成功返回账户指针，失败返回 NULL。

```cpp
Account* login(const string& role) {
    string username, password;
    cout << "请输入账号: "; cin >> username;
    cout << "请输入密码: "; cin >> password;
    // 遍历所有账户，匹配账号、密码和角色
    for (size_t i = 0; i < accounts.size(); i++) {
        if (accounts[i].username == username
            && accounts[i].password == password
            && accounts[i].role == role) {
            return &accounts[i];  // 登录成功
        }
    }
    return NULL;  // 失败
}
```

用户在登录时还会检查封禁状态：若 `bannedUntil` 大于当前时间，登录成功后会出现封禁提醒，告知封禁截止时间和剩余天数。**被封禁用户仍可登录**，但尝试下单时会被阻止。

### 2.4 用户注册

主菜单第 4 项为"注册新用户"，无需登录即可使用。注册要求提供以下信息：

1. **用户名** —— 必填，系统会检查是否已被注册
2. **密码** —— 必填，输入两次确认是否一致
3. **身份证号** —— 必填，用于下单前身份验证
4. **电话号码** —— 必填

```cpp
void userRegister() {
    string username, password, confirmPwd, idCard, phone;

    // 输入并检查用户名是否重复
    cout << "用户名: "; getline(cin, username);
    for (size_t i = 0; i < accounts.size(); i++) {
        if (accounts[i].username == username) {
            cout << "该用户名已被注册！" << endl;
            return;
        }
    }

    // 密码与确认密码
    cout << "密码: "; getline(cin, password);
    cout << "确认密码: "; getline(cin, confirmPwd);
    if (password != confirmPwd) {
        cout << "两次密码不一致！" << endl;
        return;
    }

    // 身份证号和电话号码
    cout << "身份证号: "; getline(cin, idCard);
    cout << "电话号码: "; getline(cin, phone);

    // 注册成功，加入账户列表
    Account newAcc;
    newAcc.username = username;
    newAcc.password = password;
    newAcc.role = "user";
    newAcc.displayName = username;
    newAcc.idCard = idCard;
    newAcc.phone = phone;
    // ... 初始化 cancelCount、bannedUntil 等
    accounts.push_back(newAcc);
    cout << "注册成功！" << endl;
}
```

> 新注册用户自动获得普通用户角色，可用注册的用户名和密码直接登录。下单前身份证验证功能同样适用。

## 3. 枚举与核心数据结构

### 3.1 货物类型 (CargoType)

四类货物分别对应不同的无人机等级要求：

| 类型     | 含义     | 配送要求               |
|----------|---------|-----------------------|
| Normal   | 普通货物 | 轻型以上即可            |
| Fresh    | 生鲜     | 中型以上（需温控）       |
| Medicine | 药品     | 中型以上（需温控）       |
| Valuable | 贵重物品 | 仅重型（需保险锁）       |

```cpp
enum CargoType { Normal, Fresh, Medicine, Valuable };
```

### 3.2 订单状态 (OrderState)

订单从创建到终态经历 6 个状态：

```
待处理 → 已分配 → 配送中 → 已送达 → 已收货（终态）
                                      ↘ 已取消（终态，可中途取消）
```

```cpp
enum OrderState {
    Pending,      // 待处理
    Assigned,     // 已分配
    Delivering,   // 配送中
    Delivered,    // 已送达
    Received,     // 已收货（终态）
    Cancelled     // 已取消（终态）
};
```

### 3.3 无人机等级 (DroneLevel)

| 等级             | 载重上限 | 可运货物                    | 特殊功能    |
|-----------------|---------|---------------------------|------------|
| LightDroneLevel  | ≤5kg    | 仅普通货物                  | 无          |
| MediumDroneLevel | ≤20kg   | 普通+生鲜+药品              | 温控系统     |
| HeavyDroneLevel  | ≤50kg   | 全部类型                    | 温控+保险锁  |

```cpp
enum DroneLevel { LightDroneLevel, MediumDroneLevel, HeavyDroneLevel };
```

### 3.4 Order 类

封装了订单的所有信息，包括货物、收件人、配送站点、预约时间等。

```cpp
class Order {
private:
    string orderID;          // 订单编号（如 DD20260607001）
    CargoType cargoType;     // 货物类型
    string recipientName;    // 收件人姓名
    string recipientPhone;   // 收件人电话
    string recipientAddress; // 收件地址
    double weight;           // 货物重量(kg)
    string description;      // 货物描述
    OrderState orderState;   // 订单当前状态
    string ownerName;        // 下单用户
    time_t scheduledTime;    // 预约时间戳（0=即日）
    string timeSlot;         // 时段（上午/下午/晚上）
    string fromStation;      // 起点站点代号 A/B/C/D
    string toStation;        // 终点站点代号 A/B/C/D
    int deliveryMinutes;     // 预计配送时长（分钟）
};
```

### 3.5 Drone 基类与子类

Drone 是抽象基类，定义纯虚函数 `canCarry()` 和 `getSpecialFeatures()`。三个子类分别实现：

- **LightDrone**：`canCarry` 返回 `ct == Normal`
- **MediumDrone**：`canCarry` 返回 `Normal || Fresh || Medicine`，特殊功能"温控系统"
- **HeavyDrone**：`canCarry` 返回 `true`（全部），特殊功能"温控系统 + 保险锁"

```cpp
class Drone {
protected:
    string droneID;          // 编号 L001/M001/H001
    DroneState droneState;   // 当前状态
    DroneLevel level;        // 无人机等级
    double maxWeight;        // 最大载重
    int deliveryCount;       // 当日配送次数
    string currentLocation;  // 当前位置（站点代号）
    time_t deliveryStartTime;// 配送开始时间戳
    int deliveryTotalMinutes;// 预计总时长
public:
    virtual bool canCarry(CargoType ct) const = 0;     // 纯虚
    virtual string getSpecialFeatures() const = 0;      // 纯虚
    virtual void deliver(vector<Order>& allOrders);     // 可重写
    int getRemainingMinutes() const; // 实时剩余时长
};
```

## 4. 配送地图与路线计算

### 4.1 站点定义

四个站点呈**线性排列**，相邻站点间固定 10 分钟：

```
A(天马公寓) — B(德智公寓) — C(综合楼) — D(东方红广场)
```

```cpp
const string STATION_NAMES[4] = {"天马公寓", "德智公寓", "综合楼", "东方红广场"};
const string STATION_CODES[4] = {"A", "B", "C", "D"};
const int SEGMENT_TIME = 10;
```

### 4.2 距离计算

根据两个站点的索引差值 × 10 分钟，计算配送时长。

| 路线 | 距离 | 耗时 |
|------|------|------|
| A → B | 1段 | 10分钟 |
| A → C | 2段 | 20分钟 |
| A → D | 3段 | 30分钟 |
| B → D | 2段 | 20分钟 |

```cpp
int calcDeliveryTime(const string& from, const string& to) {
    int fi = -1, ti = -1;
    for (int i = 0; i < 4; i++) {
        if (STATION_CODES[i] == from) fi = i;
        if (STATION_CODES[i] == to)   ti = i;
    }
    int dist = fi > ti ? fi - ti : ti - fi;  // |索引差|
    return dist * SEGMENT_TIME;               // ×10分钟
}
```

### 4.3 无人机定位

每架无人机有一个 `currentLocation` 字段，初始为 A（天马公寓）。每次配送完成后更新为目标站点。空闲时无人机停在四个站点之一。配送员和管理员可以看到每架无人机的当前位置。

```cpp
// 配送完成后：
currentLocation = to;         // 无人机停在目标站点
droneState = Idle;            // 恢复空闲
```

### 4.4 实时剩余时长

`getRemainingMinutes()` 用 `deliveryStartTime` 与当前时间比较，实时计算还剩多少分钟：

```cpp
int getRemainingMinutes() const {
    if (deliveryStartTime == 0 || droneState != DRDelivering) return 0;
    time_t now = time(0);
    int elapsed = (int)difftime(now, deliveryStartTime) / 60;
    int remain = deliveryTotalMinutes - elapsed;
    return remain > 0 ? remain : 0;
}
```

## 5. 订单全生命周期

### 5.1 订单编号生成

`generateOrderID()` 用当前日期 + 自增序号生成唯一编号，格式为 `DD + 年月日 + 3位序号`。

```cpp
string generateOrderID() {
    time_t now = time(0);
    tm* ltm = localtime(&now);
    stringstream ss;
    ss << "DD"
       << setw(4) << setfill('0') << (1900 + ltm->tm_year)
       << setw(2) << setfill('0') << (1 + ltm->tm_mon)
       << setw(2) << setfill('0') << ltm->tm_mday
       << setw(3) << setfill('0') << orderCounter;
    orderCounter++;
    return ss.str();
}
```

> 例如 2026年6月7日的第1笔订单：`DD20260607001`

### 5.2 用户下单流程

用户选择角色登录后，进入下单界面。下单前会依次执行以下步骤：

#### 第一步：封禁检查

如果账号已被封禁，无法进入下单流程：

```cpp
if (isBanned(*myAcc)) {
    cout << "账号已被封禁至 xxx，无法下单！" << endl;
    return;
}
```

#### 第二步：身份证验证

检查通过后，要求输入身份证号进行身份验证：

```cpp
cout << "请输入身份证号验证身份: ";
string inputId;
cin >> inputId;
if (inputId != myAcc->idCard) {
    cout << "身份证号错误！下单失败。" << endl;
    return;
}
```

> 预置账号的身份证号见[预置用户信息总表](#22-预置用户信息总表)。

#### 第三步：填写订单信息

验证通过后进入正式下单流程：

1. 选择**货物类型**（普通/生鲜/药品/贵重）
2. 输入**货物重量**
3. 选择**配送方式**（即日 / 预约1-3天）
4. 如预约则选**日期 + 时段**（上午/下午/晚上）
5. 选择**起点站点和终点站点**
6. 输入**收件人信息和详细地址**、货物描述

```cpp
// 站点选择
cout << "\n【配送地图】A=天马公寓  B=德智公寓  C=综合楼  D=东方红广场" << endl;
cout << "请选择配送起点 (1-4): ";
cin >> fromChoice;
string fromStation = STATION_CODES[fromChoice - 1];
// 同理选择终点 toStation
int delMinutes = calcDeliveryTime(fromStation, toStation);

// 创建订单
Order newOrder(oid, ct, rname, rphone, raddr, w, desc,
               Pending, userName, schedTime, timeSlot,
               fromStation, toStation, delMinutes);
allOrders.push_back(newOrder);
```

配送员在"待处理订单"中选择一笔，系统自动筛选符合条件的无人机：

选择界面还额外显示"已配送 N/5 次"，让配送员了解该无人机的当日负担。

### 5.4 配送执行

配送员选择已分配的任务后，调用对应无人机的 `deliver()` 方法：

```cpp
// deliver() 核心逻辑

string to   = order->getToStation();
```

### 5.5 用户确认收货

用户看到自己状态为"已送达"的订单，选择确认后状态变为"已收货"（终态）。

## 6. 无人机配送机制

### 6.1 每日配送上限

每架无人机有 `deliveryCount` 计数器，每完成一次配送 +1。达到上限后不再出现在候选列表中。

```cpp
const int MAX_DAILY_DELIVERIES = 5;

// 在所有 deliver() 中：
if (deliveryCount >= MAX_DAILY_DELIVERIES) {
    cout << "已达配送上限，无法继续配送！" << endl;
    order->setOrderState(Pending);  // 退回待处理
    return;
}
```

### 6.2 管理员查看当日配送记录

管理员菜单第 5 项分两部分展示：

1. **上半部分**：按轻型/中型/重型分组显示每架无人机的配送次数、当前位置、上限比例
2. **下半部分**：统一列出所有已完成订单（已送达+已收货），每笔订单只显示一次

此外还会汇总当日总配送次数。

## 7. 预约下单与取消

### 7.1 预约配送

用户可选择"即日配送"或"预约配送（1-3天）"。选预约后列出未来 3 天的日期，再选时段。

```cpp
if (modeChoice == 2) {
    time_t now = time(0);
    for (int day = 1; day <= 3; day++) {
        time_t dayTime = now + day * 24 * 60 * 60;
        tm* dtm = localtime(&dayTime);
        cout << day << ". " << (1900+dtm->tm_year) << "-"
             << (1+dtm->tm_mon) << "-" << dtm->tm_mday << endl;
    }
    cin >> dayChoice;
    schedTime = now + dayChoice * 24 * 60 * 60;
    // 选时段：上午8-12 / 下午12-17 / 晚上17-21
}
```

时段映射：
- `1` → `"上午8-12"`
- `2` → `"下午12-17"`
- `3` → `"晚上17-21"`

### 7.2 取消订单

用户可取消状态为"待处理"或"已分配"的订单（配送中/已送达/已收货不可取消）。取消后订单状态变为 `Cancelled`，并调用 `recordCancellation()` 记录。

**关键：取消已分配订单时释放无人机。** 如果取消的订单已被分配给某架无人机，必须遍历所有无人机对其中的 `assignedOrderID` 和 `assignedBy` 进行解绑，将其重置为 `"#"`，防止无人机配送一个已不存在的订单。

```cpp
// 如果是已分配状态，释放被绑定的无人机
if (allOrders[orderIndex].getOrderState() == Assigned) {
    string cancelledOrderID = allOrders[orderIndex].getOrderID();
    for (auto& d : lightDrones) {
        if (d.getAssignedOrderID() == cancelledOrderID) {
            d.setAssignedOrderID("#"); d.setAssignedBy("#");
        }
    }
    for (auto& d : mediumDrones) {
        if (d.getAssignedOrderID() == cancelledOrderID) {
            d.setAssignedOrderID("#"); d.setAssignedBy("#");
        }
    }
    for (auto& d : heavyDrones) {
        if (d.getAssignedOrderID() == cancelledOrderID) {
            d.setAssignedOrderID("#"); d.setAssignedBy("#");
        }
    }
}
allOrders[orderIndex].setOrderState(Cancelled);
recordCancellation(*myAcc);  // 记录取消次数
```

## 8. 封禁机制

### 8.1 取消计数与封禁

每周累计取消 **3次** 自动封号 **14天**。`recordCancellation()` 先检查是否跨周（以周一为界重置计数），然后 `cancelCount + 1`，达到 3 次时设 `bannedUntil`。

```cpp
void recordCancellation(Account& acc) {
    resetCancelCountIfNewWeek(acc);    // 跨周则重置计数
    acc.cancelCount++;
    if (acc.cancelCount >= 3) {
        acc.bannedUntil = time(0) + 14 * 24 * 60 * 60;
        cout << "封禁至 " << formatBanTime(acc.bannedUntil) << endl;
    }
}
```

跨周重置逻辑：计算本周一 0:00 的时间戳，若记录的 `cancelWeekStart` 早于它，则清零 `cancelCount`。

### 8.2 登录不限制，下单被阻止

封禁策略为"**能看不能用**"：

- **登录不受影响**：被封禁用户仍可登录系统，查看自己的订单记录
- **下单被阻止**：进入下单功能时主动检测封禁状态，提示用户封禁剩余天数并拒绝下单

```cpp
// 登录时：仅显示提醒，不阻止进入
if (isBanned(*acc)) {
    cout << "⚠ 账号已被封禁，无法下单！（至 xxxx-xx-xx）" << endl;
}
// 正常进入用户菜单

// 下单时：检测到封禁则拒绝
if (isBanned(*myAcc)) {
    cout << "账号已被封禁，无法下单！" << endl;
    return;
}
```

> 登录成功后会显示本周已取消次数。被封禁用户登录时还会额外显示封禁截止日期和剩余天数。

## 10. 输入健壮性与防刷屏

### 问题背景

程序中所有数字输入使用 `cin >> int_var` 读取。当用户误输入非数字字符（如字母、符号），`cin` 会进入 **fail 状态**，错误字符持续残留在缓冲区中。在 C++98/03 模式下，变量保留原值，循环无法退出，导致不断清屏刷新菜单——即无限刷屏现象。

### 解决方案

封装两个安全输入函数 `readInt()` / `readDouble()`，内部通过 `cin.fail()` 检测非法输入，一旦失败即调用 `cin.clear()` 清除错误状态、`cin.ignore()` 丢弃残余字符，并提示用户重新输入：

```cpp
int readInt() {
    int value;
    while (!(cin >> value)) {
        cin.clear();
        cin.ignore(numeric_limits<streamsize>::max(), '\n');
        cout << "输入无效，请输入数字: ";
    }
    cin.ignore(numeric_limits<streamsize>::max(), '\n');
    return value;
}

double readDouble() {
    double value;
    while (!(cin >> value)) {
        cin.clear();
        cin.ignore(numeric_limits<streamsize>::max(), '\n');
        cout << "输入无效，请输入数字: ";
    }
    cin.ignore(numeric_limits<streamsize>::max(), '\n');
    return value;
}
```

### 替换范围

所有菜单选择、数量输入、站点选择等共计 17 处 `cin >> int/double` 全部替换为 `readInt()` / `readDouble()`，从根源杜绝刷屏问题。
